# Fix Show More/Less Button on Mobile

## Problem Analysis

### Bug 1: Closes randomly on mobile
**Root cause:** `menu/menu-loader.js:84` — `window.addEventListener("resize", applyLimit)` fires `applyLimit()` on every resize event. On mobile, `resize` fires when the address bar hides on scroll, the keyboard opens/closes, or the viewport shifts slightly. `applyLimit()` (line 55-69) **unconditionally re-collapses** the grid — it never checks if the user manually expanded it. So:

1. User taps "Show More" → grid expands
2. Mobile browser fires resize → `applyLimit()` runs → grid snaps back to collapsed

### Bug 2: Scroll position jumps on toggle
**Root cause:** When "Show More" is clicked, `maxHeight` is removed (line 74) and the grid expands to full height. All content below shifts downward, pushing the user's viewport away from where they were.

## Fix (single file: `menu/menu-loader.js`)

### Change 1: Track expanded state
Add a boolean `let expanded = false;` inside `setupShowMore()`. Set it to `true` in `toggleShow()` when expanding, `false` when collapsing.

### Change 2: Guard `applyLimit()` against user-expanded state
At the top of `applyLimit()`, if `expanded` is `true`, return early — don't re-collapse what the user opened.

### Change 3: Preserve scroll position on toggle
In `toggleShow()`, capture `window.scrollY` before the DOM change and restore it after. Since removing `maxHeight` can change the document height and shift content, restore `window.scrollY` to its pre-toggle value to keep the user at the same visual spot.

### Concrete diff

```javascript
function setupShowMore(containerId, gridId) {
    const grid = document.getElementById(gridId);
    const btn = document.getElementById(containerId + "-btn");
    if (!grid || !btn) return;

    const cards = grid.querySelectorAll(".cake-card");
    if (cards.length <= 4) { btn.style.display = "none"; return; }

    let expanded = false;

    function applyLimit() {
        if (expanded) return;                        // ← NEW: skip if user expanded
        const isMobile = window.innerWidth <= 768;
        const limit = isMobile ? 2 : 4;
        if (cards.length <= limit) {
            grid.classList.remove("collapsed");
            grid.style.maxHeight = "";
            return;
        }
        const lastVisible = cards[limit - 1];
        const gridRect = grid.getBoundingClientRect();
        const cardRect = lastVisible.getBoundingClientRect();
        const maxH = cardRect.bottom - gridRect.top + 5;
        grid.style.maxHeight = maxH + "px";
        grid.classList.add("collapsed");
    }

    function toggleShow() {
        const scrollY = window.scrollY;              // ← NEW: capture position
        if (grid.classList.contains("collapsed")) {
            grid.classList.remove("collapsed");
            grid.style.maxHeight = "";
            expanded = true;                          // ← NEW
            btn.innerHTML = "Show Less <span class='btn-shadow'></span>";
        } else {
            expanded = false;                         // ← NEW
            applyLimit();
            btn.innerHTML = "Show More <span class='btn-shadow'></span>";
        }
        window.scrollTo(0, scrollY);                  // ← NEW: restore position
    }

    btn.addEventListener("click", toggleShow);
    applyLimit();
    window.addEventListener("resize", applyLimit);
}
```

Total changes: **3 lines added, 0 lines removed** — the function signature, structure, and all existing behavior remain unchanged.
